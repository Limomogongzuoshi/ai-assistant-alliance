# 知乎 OAuth 回调地址配置说明

## 关键概念

### 回调地址的作用
回调地址（redirect_uri）是 OAuth 授权流程中非常重要的一环：

1. 用户在知乎授权后
2. 知乎将用户重定向回这个地址
3. 同时在 URL 参数中带上 `code`（授权码）

### 回调地址的要求
- **必须与知乎开放平台配置一致**
- **必须是可访问的 URL**
- **能够接收 code 参数**

---

## 两种配置方案

### 方案 A：本地回调（推荐用于开发）
**配置：**
- 知乎开放平台：`http://localhost:17993/callback`
- Mock 服务：监听 `http://localhost:17993/callback`
- **优点**：可以捕获 code 进行测试
- **适用**：开发调试阶段

### 方案 B：知乎实际 URL（用于生产）
**配置：**
- 知乎开放平台：`https://www.zhihu.com/pin/2037901044210997153`
- 授权后跳转到这个页面
- **注意**：无法捕获 code，只能看到最终结果
- **适用**：验证授权流程是否成功

## 重要提示

### 为什么使用知乎 URL 作为回调？
这个知乎 URL 可能是：
1. **测试页面**：专门用于接收授权回调
2. **中间页面**：页面中有脚本处理 code
3. **代理页面**：转发 code 到其他地方

### 如何获取 code？
如果需要捕获 code 进行调试，有两种方式：

**方式 1**：使用本地回调（临时）
- 临时将知乎开放平台的回调地址改为：`http://localhost:17993/callback`
- 测试完成后改回：`https://www.zhihu.com/pin/2037901044210997153`

**方式 2**：在浏览器控制台查看
- 授权跳转时，按 F12 打开开发者工具
- 在 Network 标签页查看跳转请求
- 找到包含 code 的 URL

## 常见问题

### Q: 授权后没有跳转到回调地址？
A: 检查：
1. 知乎开放平台配置是否正确
2. APP_ID 和 APP_KEY 是否正确
3. 回调地址是否与配置一致

### Q: 跳转了但没有看到 code？
A: 可能的原因：
1. 授权被拒绝
2. APP 权限不足
3. code 已被使用或过期

### Q: 如何验证授权是否成功？
A: 可以：
1. 观察是否跳转到回调地址
2. 检查 URL 中是否有 code 参数
3. 尝试使用 code 换取 token
