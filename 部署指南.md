## 🚀 部署到 GitHub Pages（手动方式）

### 步骤 1：访问 GitHub 仓库

打开浏览器访问：
https://github.com/Limomogongzuoshi/ai-assistant-alliance

### 步骤 2：上传文件

1. 点击绿色的 **"Code"** 按钮
2. 选择 **"Open with GitHub Desktop"** 或直接点击 **"Upload files"**
3. 或者点击 **"Add file"** → **"Upload files"**

### 步骤 3：拖拽上传以下文件

需要上传的核心文件：
- ✅ `index.html` - 主页面（必须）
- ⭕ `zhihu-oauth-server.js` - OAuth 服务器（Node.js项目，不需要上传到GitHub Pages）
- ⭕ `PITCH_DECK.md` - 演讲稿（可选）
- ⭕ `项目介绍.md` - 项目介绍（可选）

### 步骤 4：提交更改

1. 在页面底部找到 **"Commit changes"** 部分
2. 添加提交信息：`Add AI Assistant Alliance demo`
3. 点击 **"Commit changes"** 绿色按钮

### 步骤 5：启用 GitHub Pages

1. 点击仓库顶部的 **"Settings"**（设置）
2. 在左侧菜单找到 **"Pages"**
3. **Source（来源）**：选择 **"Deploy from a branch"**
4. **Branch（分支）**：
   - 选择 **"main"**
   - 文件夹选择 **"/ (root)"**
5. 点击 **"Save"** 按钮

### 步骤 6：等待部署

- 等待 1-2 分钟
- 刷新 Settings 页面
- 顶部会显示绿色横幅：
  ```
  Your site is live at https://limomogongzuoshi.github.io/ai-assistant-alliance/
  ```

### 步骤 7：访问您的网站

打开浏览器访问：
https://limomogongzuoshi.github.io/ai-assistant-alliance/

## ⚠️ 注意事项

### GitHub Pages 限制
- 仅支持静态网站
- 不支持服务器端代码（如 Node.js）
- 所有功能必须在浏览器中运行

### OAuth 功能说明
- 由于 GitHub Pages 是静态托管，真实的 OAuth 需要：
  - 本地运行 OAuth 服务器，或
  - 使用后端即服务平台（如 Vercel、Railway）

### 演示模式
- 所有功能都提供「演示模式」
- 无需真实 OAuth 即可完整体验
- 适合快速演示和测试

## 🆘 常见问题

### Q: GitHub Pages 没有显示网站？
- 等待 2-3 分钟后刷新
- 检查 Settings → Pages 配置
- 确认分支选择正确

### Q: 页面显示 404？
- 确认 index.html 在仓库根目录
- 检查文件名大小写
- 确认 GitHub Pages 已启用

### Q: 功能不工作？
- 使用演示模式体验
- 打开浏览器开发者工具（F12）查看错误
- 检查网络连接
